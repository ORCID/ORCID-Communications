Streamlined permission workflow

In May we’ll be making changes to the ORCID API to improve permission request workflows. When ORCID is being used to authenticate access, we have removed repeated requests to grant access. These changes also support organizations that share access tokens to enable other services to make updates on their behalf.

The new functionality will support persistent login. If a record holder has previously granted access to their record, they will not be prompted to grant access again; instead they will be taken directly to your site. Also, if the user grants permissions multiple times, different access tokens will be issued each time, even if the same scopes (permissions) are requested. These changes are in sandbox for testing and will be available in the live Registry later this month.
